# Checklist de Qualidade — SaaS Mestre Orc

Este checklist organiza os pontos necessários para evoluir o SaaS Mestre Orc de protótipo avançado para produto full stack.

## 1. Experiência atual

- [ ] Login demo funcionando.
- [ ] Dashboard carregando corretamente.
- [ ] Gestão de campanhas funcionando.
- [ ] Gestão de personagens funcionando.
- [ ] Gestão de reservas funcionando.
- [ ] Exportação JSON validada.
- [ ] Foundry Bridge validado.
- [ ] Geração local de aventura funcionando.
- [ ] Marketplace simulado navegável.

## 2. Organização técnica

- [ ] Arquivos principais documentados.
- [ ] Funções JavaScript organizadas por responsabilidade.
- [ ] Dados salvos no LocalStorage com chave clara.
- [ ] Fluxos principais sem erro no console.
- [ ] Layout responsivo validado.
- [ ] Estados vazios tratados.

## 3. Evolução para backend

- [ ] Definir modelos MongoDB finais.

- [ ] Criar autenticação JWT.
- [ ] Criar CRUD real de campanhas.
- [ ] Criar CRUD real de personagens.
- [ ] Criar CRUD real de reservas.
- [ ] Migrar LocalStorage para banco.

- [ ] Criar painel financeiro.

## 6. Documentação

- [ ] README atualizado.
- [ ] `CASE_TECNICO.md` criado.
- [ ] `QUALITY_CHECKLIST.md` criado.
- [ ] Blueprint de API atualizado.

Este projeto deve ser apresentado como produto autoral em evolução, demonstrando visão de negócio, prototipação, dashboards, integração com ferramentas externas e planejamento para SaaS real.
